# XiAutoTags（Typecho 1.3.0 轻量重构版）

XiAutoTags 是一个给 Typecho 后台写作页用的 AI 标签插件。
本仓库当前版本为 **v2.1.0 重构版**，目标是：

- 保留核心能力（AI 生成标签）
- 删除旧版复杂且不稳定的逻辑
- 尽量轻量、可维护、可直接落地

## 功能特性

- 在文章编辑页一键触发 AI 生成标签
- 同时支持 `Chat Completions` 和 `Responses` 两种接口格式
- 生成结果以“可点击标签按钮”展示，不强制自动写入
- 三种颜色状态一眼可见：
  - 黄色：标签库里已有
  - 绿色：AI 新标签（标签库里没有）
  - 灰色：当前文章已选中
- 点击按钮即可添加到 Typecho 标签输入框
- 删除已选标签后，按钮状态会自动刷新

## 适配版本

- Typecho `1.3.0`
- 需要服务器启用 `cURL`
- 支持 OpenAI 兼容 `Chat Completions` 接口
- 支持 OpenAI 兼容 `Responses` 接口

## 目录结构

- `Plugin.php`：插件入口、后台配置、编辑页按钮与交互
- `Action.php`：后端 Action 路由，调用 AI 并返回标签结果

## 安装方法

1. 将仓库文件放到：`/usr/plugins/XiAutoTags/`
2. 在 Typecho 后台启用插件 `XiAutoTags`
3. 打开插件设置，填写 API 参数
4. 进入“写文章”页面，点击 `AI 生成标签`

## 配置项说明（精简）

- `API 端点`：如 `https://openrouter.ai/api/v1/chat/completions` 或 `https://api.openai.com/v1/responses`
- `API Key`：你的服务密钥
- `模型名称`：如 `deepseek/deepseek-chat`
- `接口类型`：自动识别 / Chat Completions / Responses
- `最多标签数`：一次最多返回多少个标签（建议 3-8）
- `最大内容长度`：发送给 AI 的正文截取长度
- `请求超时（秒）`

## 使用说明

1. 在文章页填写标题（正文可空，但建议填写）
2. 点击 `AI 生成标签`
3. 等待标签按钮出现
4. 按需点击标签添加到文章

> 说明：当前交互是“手动点击添加”，不是旧版那种自动全部写入。

## v2 重构说明

旧版代码包含大量安全策略、多 API 路由、缓存与复杂前端控制台逻辑，维护成本高且在部分环境不可用。

v2 重构后：

- 保留管理员权限校验
- 使用 Typecho Action 标准路由
- 支持 `Responses API`
- 统一为简洁可读的提示词和返回解析
- 去除多余依赖与复杂流程，优先保证可用性

## 常见问题

### 1) 点按钮没有结果
- 检查 `API Key`、`模型名称`、`API 端点` 是否正确
- 检查服务器是否启用 `cURL`
- 看浏览器网络请求是否返回错误信息

### 2) 标签颜色不对
- 颜色依据后端返回的标签库比对和当前文章已选标签实时计算
- 删除/新增标签后会自动刷新状态
